# Implementation Plan: BPM-Synced Looper

**Branch**: `015-bpm-looper` | **Date**: 2026-04-18 | **Spec**: [spec.md](spec.md)
**Input**: Feature specification from `/specs/015-bpm-looper/spec.md`

## Summary

A single-track BPM-synced audio looper with a doughnut ring canvas display. Records audio input to a `Float32Array` buffer using `ScriptProcessorNode`, loops it via `AudioBufferSourceNode` with `loop = true`, supports overdub by in-place sample accumulation, and always passes live input to the output via a permanent `GainNode`. State machine has four states (idle / recording / playing / overdubbing) driven by on-canvas buttons and keyboard shortcuts (1 / 2 / 0). The musical Keyboard module gains a reserved-key list to prevent note events on those keys. `ComponentData` is extended with an optional `audioBlob` field for loop persistence.

## Technical Context

**Language/Version**: TypeScript 5.6+, ES2020 target, strict mode
**Primary Dependencies**: Web Audio API, DOM — zero runtime dependencies
**Storage**: `localStorage` via existing `PatchSerializer` / `PatchStorage` pattern; `ComponentData.audioBlob?: string` added for Base64 PCM
**Testing**: Vitest (run via `vitest run`)
**Target Platform**: Browser (Vite dev server / static build)
**Project Type**: Single-page modular synthesizer app
**Performance Goals**: 60 FPS canvas rendering; audio parameter changes take effect within one Web Audio scheduler tick (~128 samples)
**Constraints**:
- Zero new runtime dependencies — Web Audio API + DOM only
- TypeScript strict mode enforced
- `ScriptProcessorNode` used for audio capture (no AudioWorklet — not in this codebase)
- Patch format changes must be backward-compatible (`audioBlob` is optional)

### Step 2: Looper SynthComponent

**File**: `src/components/utilities/Looper.ts`

Implements `SynthComponent` + `TempoAware`. All audio work happens inside the Web Audio API.

**Audio node graph:**

```
[input port]
     │
     ▼
[inputGain: GainNode (gain=1)]
     │
     ├──────────────────────────────► [outputMix: GainNode]──► [output port]
     │ (always-on passthrough)                ▲
     │                                        │
     ▼                               [playbackSource: AudioBufferSourceNode]
[captureNode: ScriptProcessorNode]     (loop = true, created per playback start)
     │
     └─ onaudioprocess: writes/mixes samples into Float32Array buffer
```

**Key methods:**

| Method | Description |
|--------|-------------|
| `createAudioNodes()` | Creates `inputGain`, `captureNode` (bufferSize=4096), `outputMix`. Connects passthrough. |
| `destroyAudioNodes()` | Stops `playbackSource`, disconnects all nodes. |
| `startRecording()` | Sets `state = RECORDING`. Computes `loopDurationSamples`. Allocates `Float32Array`. Arms `captureNode.onaudioprocess`. |
| `stopRecording()` | Called automatically when `writeHead >= loopDurationSamples`. Commits buffer. Calls `startPlayback()`. |
| `startPlayback()` | Creates `AudioBufferSourceNode` from `Float32Array`. Sets `loop=true`, `loopStart=0`, `loopEnd=durationSec`. Starts. Sets `state = PLAYING`. |
| `stopPlayback()` | Stops and nulls `playbackSource`. Sets `state = IDLE`. |
| `startOverdub()` | Switches `captureNode.onaudioprocess` to mix mode (`buffer[i] += input`). Sets `state = OVERDUBBING`. |
| `stopOverdub()` | Re-commits updated buffer. Restarts `AudioBufferSourceNode` from current position. Sets `state = PLAYING`. |
| `clearLoop()` | Stops playback. Nulls buffer. Sets `state = IDLE`. |
| `pressRecord()` | State guard: idle only → `startRecording()`. |
| `pressStop()` | State guard: overdubbing → `stopOverdub()` (stays playing). Playing → `stopPlayback()`. |
| `pressClear()` | Any state → `clearLoop()`. |
| `applyGlobalBpm(bpm)` | Updates `this.currentBpm`. Only affects the next recording (current loop unaffected). |
| `serialize()` | Returns `ComponentData` with `parameters: {barCount, stateIndex}` and `audioBlob` (Base64 of `Float32Array`). |
| `deserialize(data)` | Restores `barCount`, `state` (via `stateIndexToLooperState()`), and `Float32Array` from `audioBlob`. |

**Quantised record start**: `startRecording()` reads the current beat position from `globalBpmController` and delays buffer allocation by the remaining samples until the next beat boundary using `AudioContext.currentTime` scheduling.

### Step 3: LooperDisplay

**File**: `src/canvas/displays/LooperDisplay.ts`

Follows `ColliderDisplay` pattern — own `HTMLCanvasElement`, appended to DOM alongside the main canvas.

**Rendering per frame (called from `CanvasComponent` render loop):**

1. `clearRect` full canvas.
2. `drawRing(state)` — filled arc, full circle, colour from `LOOPER_STATE_COLORS[state]`, ring width ~8 px.
3. `drawPlayhead(playHeadNormalized)` — thin line from ring centre to outer edge at angle `playHeadToAngle(normalized)`. Only drawn when `filled = true`.
4. `drawLabel(barCount)` — centre text showing bar count (e.g. "2").
5. `drawButtons()` — four small labeled circle buttons: "R" (record), "■" (stop), "OD" (overdub), "✕" (clear) along the bottom. Bar count selector (1/2/4/8) rendered as tap targets around the ring.

**Public API:**

| Method | Description |
|--------|-------------|
| `constructor(x, y, w, h, looper)` | Creates canvas, sets initial position. |
| `getCanvas(): HTMLCanvasElement` | Returns the canvas element for DOM attachment. |
| `render(state: LooperDisplayState)` | Draws one frame. Called from CanvasComponent. |
| `updatePosition(x, y)` | Repositions canvas element. |
| `destroy()` | Removes canvas from DOM. |
| `handleMouseDown(x, y): 'record' \| 'stop' \| 'clear' \| 'overdub' \| BarCount \| null` | Hit-test button and bar count selector areas. Returns action string, selected bar count, or null. |

### Step 4: CanvasComponent Wiring

**File**: `src/canvas/CanvasComponent.ts`

Follows the Collider pattern exactly:

1. In `createControls()`: instantiate `LooperDisplay`, append its canvas to DOM.
2. In `render()`: call `looperDisplay.render(looper.getDisplayState())`.
3. In `handleMouseDown()`: delegate to `looperDisplay.handleMouseDown()`, map result to `looper.pressRecord()` / `pressStop()` / `pressClear()`.
4. In `destroy()`: call `looperDisplay.destroy()`.
5. Wire `updatePosition` / `updateSize` / `updateViewportTransform` to `looperDisplay`.

### Step 5: Keyboard Shortcut Wiring

**File**: `src/keyboard/KeyboardController.ts`

1. Add at class level:
   ```typescript
   private static readonly RESERVED_KEYS: ReadonlySet<string> = new Set(['1', '2', '0']);
   ```
2. In `handleKeyDown()`, immediately after the input-field guard:
   ```typescript
   if (KeyboardController.RESERVED_KEYS.has(e.key)) return;
   ```
3. Add a separate `window.addEventListener('keydown', ...)` in `main.ts` (or a new `LooperKeyboardHandler`) that maps `1` → `looper.pressRecord()`, `2` → `looper.pressStop()`, `0` → `looper.pressClear()`.

### Step 6: PatchSerializer Wiring

**File**: `src/patch/PatchSerializer.ts`

The `Looper.serialize()` and `deserialize()` methods handle all Looper-specific logic. PatchSerializer calls the generic `component.serialize()` / `component.deserialize()` — no changes needed to PatchSerializer itself. The `audioBlob` field passes through automatically as part of `ComponentData`.

---

### Step 7: main.ts Wiring

**File**: `src/main.ts`

1. Import `Looper` and register with `ComponentType.LOOPER` in the component factory.
2. Register the keyboard shortcut handler after `KeyboardController` is initialized.
3. Call `looper.subscribeToGlobalBpm()` on activation (handled inside `Looper.activate()`).
